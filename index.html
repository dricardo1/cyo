Introduction

Inspired by those ol'-fashioned, choose-your-own adventure books, CYO (Choose Your Own) is a declarative framework for creating the same kind of experience on the web, with added features that you just couldn't get right in the old paperback books.

CYO is used to make choose your own adventure games. You don't need to know any JavaScript at all to use it.

The Choose Your Own Adventure Engine is built with Angular.js, and requires only HTML to use. A CYO story will run in any browser, and the styling can be fully customized with CSS.

Getting Started

CYO is a nearly all-directive library. To gain access to the directive, simply require the module as a dependency of your angular app.

Using the above script, any HTML bootstrapped to your "gameApp" module will have the following useful directives added.

Story Tag

To initialize a story, you use the storytag. The story tag has an attribute that points to the first chapter of your story, usually "intro". 


          <script src='lib/angular/angular.min.js'></script>
          <script src='lib/underscore.js'></script>
          <script src='lib/underscore.string.js'></script>
          <script src='lib/jquery.js'></script>
          <script src='lib/cyo.js'></script>
          <script>
              angular.module('gameApp', ['cyo'])
          </script>
          <body ngApp='gameApp'>

Choice Tag

The story tag is pretty boring on its own. Things get a lot more interesting when you add a choice tag or two. Choices allow you to load a new page and keep the story going. When you click a choice, all the other choices on that page disappear. Let's update the intro to this story. 
    
    <page>
        <p>
          It was a very short story, and nothing at all happened.
        </p>
        <p>
          But, all of a sudden, you saw a strange man and he danced away down a grassy path.
        </p>
        <choice follow-him>
          Follow that dancing man!
        </choice>
        <choice stay-put>
          Stay put.
        </choice>
    </page>
You will now need a new file for each of those choices.
